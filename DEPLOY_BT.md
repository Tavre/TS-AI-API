# 宝塔面板 (BT Panel) 部署指南

本指南将帮助你在安装了宝塔面板的 Linux 服务器上部署 TS-AI API 文档。

## 1. 准备代码

### 方法 A：本地上传 (简单)
1. 在本地将整个 `TS-API-DOC` 文件夹打包为 `zip` 压缩包。
2. 登录宝塔面板，进入 **【文件】** 菜单。
3. 选择一个目录（如 `/www/wwwroot/`），点击 **【上传】** 并解压。

### 方法 B：Git 克隆 (推荐)
如果服务器已安装 Git，推荐直接拉取代码，方便日后更新。
1. 登录宝塔面板，进入 **【终端】**。
2. 进入网站根目录：`cd /www/wwwroot/`
3. 拉取代码：`git clone https://github.com/你的用户名/TS-API-DOC.git`

## 2. 添加站点

1. 登录宝塔面板，点击 **【网站】** -> **【添加站点】**。
2. 填写信息：
   - **域名**: 填写你的文档域名（如 `doc.example.com`）。
   - **备注**: TS-AI API 文档。
   - **根目录**: 选择刚才上传/克隆的目录（如 `/www/wwwroot/TS-API-DOC`）。
   - **FTP/数据库/PHP**: 均选择 **“不创建”** 或 **“纯静态”** (Docsify 不需要 PHP 和数据库)。
3. 点击 **【提交】**。

## 3. 设置运行目录 (关键)

Docsify 的核心文件在 `docs` 子目录下，我们需要调整网站配置指向它。

1. 在网站列表中，点击刚才创建的网站名（打开设置窗口）。
2. 点击左侧 **【网站目录】**。
3. **运行目录**：在下拉框中选择 `/docs`。
4. 点击 **【保存】**。

## 4. 伪静态配置 (可选)

Docsify 默认使用 Hash 模式（URL 中带 `#`，如 `/#/user-balance`），这在所有服务器上都能直接运行，**无需**额外配置伪静态。

如果你将来开启了 `routerMode: 'history'` 模式，则需要添加以下 Nginx 伪静态规则：

```nginx
location / {
  try_files $uri $uri/ /index.html;
}
```

## 5. 访问验证

在浏览器访问你设置的域名。
- 成功：看到 "TS-AI 开发者 API" 首页。
- 失败 (403 Forbidden)：检查【网站目录】是否正确指向了 `/docs`，且目录权限为 `755`，所有者为 `www`。
- 失败 (404 Not Found)：检查 `docs` 目录下是否有 `index.html`。
